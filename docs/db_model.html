Table session

Columns
This is the session table. It holds the drone and the user sessions to keep track of who's logged in and who's logged out.
* session_id int AUTOINCREMENT Auto Increment session identifier, keeps track of the number of active sessions.
* token varchar( 255 ) Keeps track of the active session tokens.
user_id int This is the fk that keeps track of the user_id's. Either this one or the drone_id is mandatory.
drone_id int This is the fk that keeps track of the drone_id's. Either this one or the user_id is mandatory.
Indexes
pk_session ON session_id
pk_session_0 ON user_id
pk_session_1 ON drone_id


Table user

Columns
This is the table that handles all the data for the users that are to use our mobile app to order tacos.
* user_id int AUTOINCREMENT The user_id uniquely identifies Users. Its an AutoIncrement field meaning that on user creation it need not be specified.
* facebook_id varchar( 255 ) This field stored the value of the facebook id for the user.
* first_name varchar( 35 ) Stores the first name of the user.
* last_name varchar( 35 ) Stores the last name of the user.
* email varchar( 100 ) Here is the email of the user.
Indexes
pk_user ON user_id
Foreign Keys
fk_user ( user_id ) ref session (user_id) This foriegn key contains the value of the user_id from the user database.
fk_user_0 ( user_id ) ref taco_request (user_id) Foriegn Key for the user_id in the taco requests database.


Table drone

Columns
Here is the table defining the attributes for a drone.
* drone_id int AUTOINCREMENT Here is the drone identifier. Its an Auto Increment field so it doesn't need to be included in the creation of a new drone entry.
* password varchar( 255 ) Here is the access password for the drone, to make it autonomously deliver.
* location_id int Here is the foriegn-key of the location. Defining the Area that the drone is able to deliver to.
Indexes
pk_drone ON drone_id
pk_drone_0 ON location_id
Foreign Keys
fk_drone ( drone_id ) ref session (drone_id) This foriegn key refers to the drone_id from the drone DB.
fk_drone_0 ( drone_id ) ref fulfilled request (drone_id) Here is the foriegn key for the drone object

Table location

Columns
This is the location table. It contains the coordinates that the drone can fly to.
* location_id int AUTOINCREMENT Here is a foriegn-key defined for the location. Its an identifier and it has Auto-Increment capabilities.
  location_name varchar( 100 ) Here is the name of the location specified.
Indexes
pk_location ON location_id

Table taco_request

Columns
All the user requests to deliver tacos are kept in this database.
* taco_request_id int AUTOINCREMENT This is a unique identifier of the taco request. It helps uniquely identify taco requests so that the drone can deliver tacos more effectively.
* longitude float( 20, 17 ) This is the longitude of the delivery.
* latitude float( 20, 17 ) This is the latitude for the delivery.
* created timestamp This is the timestamp that the request was created, for delivery purposes.
* taco_type enum This is the taco_type enumerated type.
* user_id int This is a foriegn key that links to the user database.
Indexes
pk_taco_request ON taco_request_id
pk_taco_request_0 ON user_id
Foreign Keys
fk_taco_request ( taco_request_id ) ref fulfilled request (taco_request_id) Here is the foriegn_key for the taco_request.


Table fulfilled request

Columns
Here is the table when the drone fulfills the delivery request.
* fulfilled_request_id int AUTOINCREMENT Here is the unique identifier for fulfilling the request. It is an Auto Increment field meaning that it doesn't need to be included in the request.
  status enum Here is the status of the taco delivery of the drone
* taco_request_id int Here is the foriegn key of taco_request.
* drone_id int Here is the foriegn key for the drone that delivered the particular taco.
Indexes
pk_fulfilled request ON fulfilled_request_id
pk_fulfilled request_0 ON taco_request_id
pk_fulfilled request_1 ON drone_id


Table taco

Columns
Taco DB, to enter the amount of tacos avaliable to deliver.
* taco_id int AUTOINCREMENT Here is a unique_id for the taco. This member is Auto-Increment.
* taco_type enum Here is the enumerated type of tacos listed.
* number int Here is the number of the type of tacos avaliable.
* created datetime Here is the date that the taco entry was created. It stores the date that the tacos were made.
* location_id int This is the location where the tacos are located. It is a foriegn key linked to the location DB.
Indexes
pk_taco ON taco_id
pk_taco_0 ON location_id
